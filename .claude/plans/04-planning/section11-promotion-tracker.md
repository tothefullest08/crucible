   - 후보 생성 → Evaluator 점수 → 자동 판정 (≥0.80 자동 승격 / ≤0.40 자동 기각 / 회색지대 수동 승격) → 사용자 확인 UX (y/N/e/s 키 매핑) → 저장 → 거부 이력 기록

## 10. 메타 — 승격 게이트의 승격 게이트

§11 승격 태스크 자체가 "승격 게이트" 대상 (자기 참조 구조). 각 승격 태스크는 다음 2단 검증 필수:

1. **1단**: 본인이 게이트 기준을 충족했다고 판단 → Evaluator 서브에이전트 1회 검토 (회의적 튜닝 · fresh context) → 통과 시 v2 diff 생성
2. **2단**: 유저 판단 필요 항목이 있다면 해당 항목에 대해 AskUserQuestion → 응답 반영 후 최종 diff 커밋

이 2단 검증을 통과하지 못한 §11 항목은 v2 정식 섹션으로 이동 **금지**. 임시 §11 블록에 남긴 채 구현 주차 진입 불가.
