### T-W3-05 — Ambiguity Score Gate (0.2 임계, 4h)

**경로**: `scripts/ambiguity-gate.sh` (신규)

**목표**: ouroboros Ambiguity Gate (포팅 자산 #20) 구현. `/plan` Phase 1 시작 시 호출. 요구사항 모호도 ≥ 0.2면 reject + `/brainstorm` 재안내. < 0.2면 pass.

**입력**:
- `$1`: requirements.md 경로

**동작** (MVP 스텁 — 수치 휴리스틱):
1. requirements.md 본문에서 모호 키워드 카운트 (`미정`, `TBD`, `나중에`, `모호`, `as needed`, `적당히`)
2. frontmatter `decisions` 배열 길이 (0이면 ambiguity ↑)
3. Scope Included/Excluded 명확도 (둘 다 있으면 ambiguity ↓)
4. 점수 계산: `score = (vague_keyword_count / 총 문단 수) + (decisions 0일 때 +0.2) + (Scope 한쪽만 있으면 +0.1)`
5. 0.0~1.0 클램프
6. stdout: `{"score": 0.XX, "verdict": "pass"|"reject", "reason": "..."}`
7. 임계 0.2: score < 0.2 → pass, score ≥ 0.2 → reject

  - `req-low-score.md` (점수 ~0.15, pass 기대)
  - `req-high-score.md` (점수 ~0.25, reject 기대)
