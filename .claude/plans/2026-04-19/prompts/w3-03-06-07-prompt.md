### T-W3-03 — YAML frontmatter 스키마 (템플릿 파일, 6h)

**경로**: `skills/plan/templates/plan-template.md` (신규)

**목표**: `/plan` output 파일의 frontmatter 구조를 명시하는 템플릿. ouroboros Seed YAML(#21) 스키마를 직접 포팅.

**템플릿 내용** (frontmatter 필수 필드):
```yaml
---
# plan.md frontmatter 스키마 (ouroboros Seed YAML 기반, v3 Dec 10)
goal: "{한 줄 목표, ≤ 120자}"
constraints:
  - "{제약 1}"
  - "{제약 2}"
AC:    # Acceptance Criteria
  hard:    # 하드 게이트, 릴리스 차단 기준
    - "{기준 1 (측정 가능)}"
    - "{기준 2}"
  stretch:    # 미달 시 2차 연기
    - "{기준}"
evaluation_principles:    # weight 합 = 1.0 필수 (assertion)
  - name: "correctness"
    weight: 0.4
    description: "{설명}"
    metric: "{측정 방법}"
  - name: "clarity"
    weight: 0.3
    description: "..."
    metric: "..."
  - name: "maintainability"
    weight: 0.3
    description: "..."
    metric: "..."
exit_conditions:
  success: "{성공 종료 조건}"
  failure: "{실패 종료 조건}"
  timeout: "{시간 한도}"
parent_seed_id: "{연결된 상위 plan의 id, 없으면 null}"
slug: "{a-zA-Z0-9_-, 화이트리스트}"
date: "{YYYY-MM-DD}"
---

# {goal}

## Phase 1: Intake
...

## Phase 2: Decomposition
...

(T-W3-02의 Phase 1~5 템플릿 참조)

## Evaluation Principles
(frontmatter evaluation_principles 자동 요약 렌더링)

## Exit Conditions
(frontmatter 기반)
```

**Weight 합 assertion 스크립트** (`skills/plan/templates/validate-weights.sh`):
- 입력: plan.md 경로
- 처리: yq로 evaluation_principles[].weight 추출 후 합산
- 출력: 합 == 1.0 ± 0.01 → exit 0, 아니면 exit 1 + 어떤 값인지 stderr
- shellcheck 통과

**검증**:
- 템플릿 YAML 파싱 통과 (`yq eval '.' plan-template.md`)
- 모든 스키마 필드 존재 (grep으로 각 필드 이름 카운트)
- validate-weights.sh로 예시 plan 검증 (유효/무효 각 1개)

1. description 또는 goal을 입력으로 받아 slug 후보 생성

  - `"Fix login bug"` → `fix-login-bug` (pass)
  - `"한국어 제목"` → unicode 제거 후 slug (pass or adjust logic)
  - `"../../../etc"` → reject (brainstorm slug-validator 결과)
  - `"title; rm -rf"` → reject
  - `""` (빈 입력) → reject
